Introducere
In cadrul materie Tehnologii Web echipa formata din Manaila Andrei si Danila Sergiu-Stefan din grupa A7 au relizat o aplicatie web pornind de la urmatoarea cerinta:

 Sa se conceapa o aplicatie Web care gestioneaza colectii de jucarii confectionate din diverse materiale, inclusiv realizeaza managementul proprietatilor aferente: categorie, valoare, dimensiuni, imagini, destinatari (copii mici, (pre)scolari, adolescenti), stare de functionare si altele.
 Sistemul va oferi suport pentru activitati de cumparare/licitare a acestor jucarii, plus administrarea utilizatorilor.
 Se vor genera statistici diverse ce pot fi exportate in formate deschise â€“ minimal, CSV si PDF.
 Se va realiza, de asemenea, un clasament al celor mai populare produse, disponibil si ca flux de date RSS.

Ce poate sa faca aplicatia noastra web
  Utilizatorii pot cauta, filtra rezultatele si pot vedea jucariile si recenziile acestora din aplicatie.
  Exista posibilitatea de a obtine un clasament al celor mai populare produse, disponibil si ca flux de date RSS.
  Jucariile pot fi plasate in cos din pagina principala, pagina de produse si pagina produsului, iar mai departe din cos poate fi plasata comanda.
  Daca Utilizatorii opteaza pentru a isi face cont acestia au avantajul de a adauga produse in lista de favorite, de a posta recenzii la jucarii si de a vedea istoricul comenzilor.
  Utilizatorii cu cont isi pot schimba date, inclusiv parola.
  Administratorul aplicatiei poate sterge recenzii, poate vedea toate comenzile si poate obtine statistici in format CSF si PDf a recenziilor si a comenzilor.
Impartirea paginilor si a functionalitatilor
Danila Sergiu-Stefan
  • header aplicatie
    • searchBar: cautare de catergorii de produse si produse in baza de date
      • printr-un event listener ("keyup"): la fiecare tasta apasata in searchBar se apeleaza o functie care actualizeza lista de categorii si face o cerere Ajax pentru a obtine lista de produse cautarii curente
  • pagina principala (main)
    • slider automat: implementat doar prin html si css
    • selector categorie: implementat doar prin html si css
    • top produse: generare de cod html din php
    • rss feed: generare xml prin php
  • pagina produse: lista de produse este generata dinamic fara a incarca pagina
    • filtrare produse
      • la fiecare modifcare a filtrelor se schimba URL-ul paginii ( history.replaceState() ) cu noile filtre dupa care se genereaza dinamic lista produselor printr-un request catre back-end prin Ajax
      • tot prin Ajax se afla noul numarul de produse a filtrarii pentru a reseta paginarea produselor
    • paginare produse: se pot ordona produsele, seta numarul de produse pe pagina, vedea numarul de produse , numarul pagiini curente si numarul de pagini
      • la schimbarea numarului de produse pe pagina sau a paginii se schimba URL-ul paginii ( history.replaceState() ) cu noua paginare dupa care se genereaza dinamic lista produselor printr-un request catre back-end prin Ajax
  • pagina produs: se poate naviga prin pozele produsului, se poate adauga in cos, in favorite ( daca esti logat), se poate naviga prin taburile 'Descriere, Specificatii, Recenzii(n)'
    • afisare nume produs pret: generare cod html prin php
    • navigarea prin poze: realizata prin JavaScript
    • nelogat: adaugarea in cos : se adauga in Local Storage produsul
    • logat: adaugarea in cos sau favorite: se face un request la back-end prin Ajax pentru a adauga produsul in baza de date
    • navigarea prin taburi: realizata prin JavaScript
        Afisare Recenzii:
      • Initializare: se adauga recenzii in tab pana cand inaltimea paginii este mai mare decat inaltimea ecranului
      • event listener pentru scroll pentru a adauga recenzii de fiecare data cand se ajunge la limita scrollului de jos: se apeleaza o functie JS Ajax pentru a a adauga recenzii la tabul de recenzii
      • event listener pentru resize pentru a reinitializeaza tabul de recenzii: se adauga recenzii pana cand inaltimea paginii este mai mare decat inaltimea ecranului
        Adaugare Recenzii (user logat):
      • dupa introducerea textului si apasarii butonului de postare se reinitializeaza tabul de recenzii pentru a afisa si recenzia nou postata
  • cos produse: se pot vedea produsele, modifica cantitate produse, sterge produse din cos si poate finaliza comanda
    • nelogat: produsele sunt salvate in Local Storage
      • pentru lista de produse din Local Storage se face un request prin Ajax pentru afisare produselor
      • pentru modifcare cantitate sau stergere produse se modifca datele din Local Storage
    • logat: in caz ca user adauga produse in cos cand nu e logat si dupa acesta se logheaza, atunci produsle salvate local vor fi transferate user-ului alatrui de celalalte produse din cos a user-ului salvate in baza de date
      • in caz ca exista produse salvate local: printr-un request Ajax se transfera lista de produse din Local Storage in baza de date, in tabaela produse_cos
      • prin alt request Ajax se afiseaza produsele user-ului din tabaela produse_cos
      • pentru modifcare cantitate sau stergere produse se face un request Ajax pentru modificare datelor din baza de date
  • pagina admin: poate sterge recenzii, poate vedea toate comenzile si poate obtine statistici in format CSF si PDf a recenziilor si a comenzilor.
    • afisare recenziilor se face ca in pagina de produs, doar ca tabul de recenzii are o inaltime fixa, cu un scroll dedicat tabului de recenzii
    • afiasre comenzilor se face prin includerea in pagina a unui fisier php care genereaza cod html
